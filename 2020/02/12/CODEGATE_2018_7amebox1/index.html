CODEGATE 2018 7amebox1

ANALYZE

flag - > 플래그

mic_check.firm -> 에뮬을 통해 구현한 프로그램
vm_name.py -> 초기 설정 스크립트

분석은 조금 어렵지만, 그 이후는 간단하게 해결할 수 있다.

디스어셈블러를 구현하면 firmware 코드를 이해할 수 있다.

귀찮아서 대충 알아볼 수 있게만 해놨다.

ASSEMBLY CODE

EXPLOIT

샾 치고 메모해준 부분이 취약점이다. 잘 분석해보면 0x12345가 카나리로써 배치되는걸 확인할 수 있다.

스택 주소는 고정이므로, 에뮬레이터 코드를 수정하여 debug.py 를 제작하면 입력값이 배치되는 스택주소를 확인할 수 있고, 에뮬레이터 opcode에 맞추어 쉘코드를 만들고, pc를 덮을 수 있으니 쉘코드쪽으로 주소를 돌리면 플래그를 얻을 수 있다!
