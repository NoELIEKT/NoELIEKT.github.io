fsop - basic

기본적인 fsop. fake _IO_FILE 구조체를 만들어 vtable의 조작된 함수 포인터가 실행되게 만들면 된다.

fsop는 일반적으로 파일 스트림을 가키리는 포인터를 fake _io_file structure로 돌려 조작된 vtable의 함수 포인터가 실행되도록 만드는 기법이다. stdout을 덮는 방법도 있지만, 나중에 포스팅 하겠다.

open

내부구조를 보면 file structure에서 공부했던 내용 그대로 나와있다.

fileno를 보면 앞서 공부한대로 파일 디스크립터가 3에 할당이 된 것을 알 수 있다. (0, 1, 2는 각각 stdin, stdout,stderr에 이미 할당이 되어있기에 그 다음 번호인 3에 할당이 된 것이다.)

마지막 0x7ffff7dd06e0은 vtable의 주소이다. 0x6020f0은 널포인터지만 참조될 때 쓰이기 때문에 그대로 맞춰주어야 한다. 나머지는 더미값으로 채워넣으면 된다.
