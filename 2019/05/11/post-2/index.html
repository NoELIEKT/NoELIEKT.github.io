File Structure

Author: PUEL Date: May 11, 2019  22:28:49 Category: pwnable

_IO_FILE_plus는 파일 스트림에서 쓰이는 구조체이다.

먼저 _IO_FILE file을 보면

_flags - > 파일 스트림의 속성을 나타냄(읽기 전용, 쓰기 전용 등등..)

_IO_read_ptr, _IO_read_end, _IO_read_base -> 읽기 버퍼에 사용

_IO_write_ptr,_IO_write_base,_IO_write_end -> 쓰기 버퍼에서 사용

IO_buf_end, _IO_buf_base - > 예비 버퍼

chain -> 다음 파일 구조체를 가리키는 포인터이다.

_fileno -> open한 파일 디스크립터의 번호. (stdin의 경우 0이 기록된다.)

  • note

    파일 디스크립터는 close() 될 경우 닫힌 번호에 우선적으로 다음 파일 디스크립터가 할당된다.
    close(2)를 할 경우 다음으로 open된 파일 디스크립터는 2에 할당된다.

파일 관리에 사용되는 함수 포인터들을 모아둔 구조체이다.
