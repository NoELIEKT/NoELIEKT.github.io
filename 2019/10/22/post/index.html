HITCON CTF 2016 BABYHEAP

  • CTF

ANALYZE NOTE

  • MENU
    • NEW
    • DELETE
    • EDIT
    • EXIT
  • NEW
    • 0x18 크기의 힙을 생성한다.
    • 사이즈를 입력받아 헤더에 저장한다.
    • 해당 사이즈로 힙2(content)을 할당한다.
  • DELETE
    • 플래그를 체크해서 오직 한번만 FREE할 수 있다.
    • CONTENT -> HEADER 순으로 FREE한다.
  • EDIT
    • 오직 한번만 edit 가능
    • header에 저장된 size만큼 header+2(header->content)에 저장한다.
  • EXIT
    • 나갈건지 말지 다시한번 결정가능 (SCANF로 읽어들임)

VULN

  • OFF- BY - ONE

EXPLOIT

  • off - by - one으로 힙안에 있는 힙 포인터를 nullbyte로 overwrite 할 수 있다.
  • overwrite된 힙 포인터가 가리키는 주소는 scanf에 의해 할당된 힙 내부인데, 거기다가 조작된 주소-0x10에 fake chunk를 만들어준다.
  • fake chunk를 free시키면 chunk overlapping이 가능해지는데, 이걸 이용해 오버랩된 청크에 있는 got를 printf의 plt 로 바꾼다.
  • 포맷 스트링 버그로 leak & 다시 got를 덮어 system으로 쉘 따면 된다.
